O que é versionamento de código.

Existem dois tipos de sistemas de controle de versão, centralizados e distribuídos.

Os centralizados possuem um único servidor onde esse possui todas as informações referentes ao projeto.

Já os distribuídos possuem uma cópia do servidor principal , onde essas cópias são distribuídas para cada usuário envolvido do sistema.

Criação de repositório

Existem duas formas de obter um repositório:

  1. Transformando um diretório local , uma pasta que não está sob controle de versão.
  1. Clonando um repositório existente.

Levando para a prática, temos os seguintes esquemas de passos a serem seguidos;

  • Criando do zero.
    • Crie uma pasta onde será o ponto de partida.
    • Após criada , acesse o Git Bash dentro dessa pasta.
    • utilize os comandos Git init para transformar em um repositório Git.
  • Navegar dentro do Git Bash
    • Utilize comandos próprios de um prompt de comando para navegar.
  • Clonar repositório
    • Dentro da pasta que deseja clonar , utilize o comando Git clone e o link do repositório.
    • Não se esqueça de verificar os token para conseguir clonar este.
  • Clonando apenas uma Branch
    • Utilizando o mesmo comando de clonagem, Git clone URL, inserimos logo em seguida as seguintes informações - - branch feature-1 - - single-branch.
    • No lugar de feature-1 colocamos o nome da branch que deseja ser clonada.

Salvando alterações

Após vermos a criação de repositórios, fica evidente que é preciso salvar ele caso tenha alguma alteração nos arquivos.

Para realizar o commit dessas alterações, são necessárias duas etapas bastante tranquilas.

  • Sinalização

    Quando desejamos realizar o commit, devemos sinalizar as pastas que iremos realizar esse commit. Através do comando Git add < files > , deixamos separados os diretórios que serão salvos.

  • Commit

    Após separar, vem a parte tranquila , através do comando Git commit , você realiza o commit das pastas selecionadas. A título de curiosidade, também é possível realizar comentários sobre o commit , utilizando -m “comentário” logo após o comando anterior, ficando assim :

    Git commit -m ”comentário“.

Desfazendo alterações

Caso tenha um Git init em uma pasta errada e queira remover, basta realizar o seguinte comando:

  • rm -rf .git

Esse comando irá remover recursivamente ou a força a pasta .git do seu diretório.

Porém caso deseje retornar a um commit anterior, desfazendo commits que vieram a ser feitos após, temos três opções de comandos.

  • Git reset - -soft

    Quando utilizados, os commits anteriores ao selecionado, são transferidos para a área de preparação, ou área de sinalização.

  • Git reset - -mixed

    Já quando utilizamos o mixed , os arquivos não são retornados para a área de sinalização, e sim para a árvore de commits, onde terá que ser sinalizado novamente.

  • Git reset - -hard

    No caso do hard , os arquivos são apagados por completo, não aparecendo tanto na árvore de commit, quando na área de preparação.

Para dar continuidade , basta dar um Git log para ver o histórico de commits e copiar o head do commit.

Também é possível utilizar o reset para remover arquivos da área de preparação.

Trabalhando com Branches

Criando uma nova branch

Para criar uma nova branch, basta usar o seguinte código.

  • Git checkout -b <Nome da branch>

Navegando entre branches

Para navegar chega a ser até mais simples, basta utilizar :

  • Git checkout <Nome da branch que deseja ir>

Mesclagem de branches

Caso todas as alterações estejam corretas em sua branch teste , realizar a mesclagem para que a branch master também possua suas alterações , chega a ser bastante intuitivo, basta acrescentar:

  • Git merge <Nome da sua branch secundária>

Exclusão de branch

Caso já tenha sido realizado todos os testes em uma branch secundária , e deseja exclui-la, basta usar o seguinte código:

  • Git branch -d <Nome da branch que deseja excluir>
