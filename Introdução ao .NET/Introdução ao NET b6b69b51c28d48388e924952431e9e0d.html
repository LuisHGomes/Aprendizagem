Introdução ao .NET

É uma plataforma de desenvolvimento unificado que permite a construção de sistema e aplicações, em outras palavras, é uma plataforma que fornece bibliotecas para que se possa desenvolver aplicações para diversas plataformas , por exemplo :

  • Desktop
  • Web
  • Cloud
  • Mobile
  • Gaming
  • IoT
  • AI

Então caso queira construir alguma aplicação em web , utiliza-se o ASP.NET , caso queira desenvolver uma aplicação em nuvem (cloud) , utiliza-se o azure juntamente com o ASP.NET .

.Net vs .Net Framework

Quando falamos dessas duas ferramentas, logo fazemos uma associação entre eles justamente devido ao nome, e em tese não estaria errado, a não ser pelo fator exclusividade. O .NET Framework é uma ferramenta legada, de uso exclusivo de sistemas Windows, enquanto o .NET, sendo uma versão atualizada , não está exclusiva apenas para sistemas do seu antecessor, podendo rodar em Linux e Mac.

Compilador do .NET

O esquema de compilação do .NET chega a ser bastante simples visto de fora, o seu código passa por um compilador de C# , indo diretamente para o que chamam de IL code ( código de linguagem intermediaria ) , um intermediador independente de seu tipo de sistema onde está sendo executado, este gera arquivos .exe e .dll . A partir desse , com os arquivos já compilados, passamos por uma fase denominada RunTime , onde possuímos o IL code , JIT compiler ( compilador Just In Time ) e o código nativo. Resumindo de forma bem juvenil , os arquivos compilados pelo IL code , ficam esperando o código ser executado para ai sim , esses passarem pelo JIT compiler e serem transformados em código nativo.

Transpilador .NET

É a conversão de uma linguagem ou implementação para outra , como um google tradutor faz com os idiomas. Ao contrário do compilador, onde a linguagem de alto nível é convertida para baixo nível , o transpilador converte exclusivamente de alto para alto nível.
